Implementation approaches

Hub-first · Watch under Execution · Hubs+shell only. Pick the build strategy.

A

Compose existing desks (recommended)

Extract shared tab chrome + action board polish. Wire real WatchControls+WatchBoard into Execution. Wire real Picks into Radar. Redirect /watch → /live?mode=watch. Fastest path, lowest risk of breaking analyze/ticket APIs.

B

Shared primitives first

Build DeskTabs, DeskToolbar, SymbolBoard, StatusStrip as design-system pieces, then re-skin hubs. More polish and consistency, longer before anything ships.

C

IA / redirects only

Fix routes and stubs with almost no visual redesign. Ships quickest but misses “professional UI / operator feel” goals.

Why A fits this pass

  • Reuses LiveDesk / Watch / Options already working
  • Fixes dead Radar stubs immediately
  • Room for board polish + security on touched APIs
  • Matches “hubs + shell only” scope

Tradeoffs

  • Some visual inconsistency may remain vs full primitives system
  • Not a full Command redesign (by design)
